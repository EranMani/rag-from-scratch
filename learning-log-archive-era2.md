## Era 2 Summary: Foundation & Graph Assembly (Commits 01–20)

**Commits 01–06** establish security and data persistence. Auth guard-clause enforcement (C01) closes unauthorized ingest paths via asyncio blocking-I/O pattern. Misspelled identifier cleanup (C02) confirms naming hygiene at day zero. Conversation history injection (C03) separates retrieval (query-pure) from context (LLM-injected post-retrieval), with an accepted permanent gap: LLM cache key excludes history to allow cache reuse across turns. User profiles (C04–C06) add SQLite schema with WAL mode, FK cascade deletion, and JWT-gated profile API. Registration is the creation event; profile creation decouples from retrieval. Static 404 details prevent CWE-209 username enumeration.

**Commits 07–10** design and assemble the LangGraph orchestration layer. AgentState (C07) is the per-session contract: add_messages reducer for history, Literal enforcement on mastery_level and cache_hit flags, and a __future__ import gotcha documented. Retrieval node (C08) detects Chroma backend availability without signature mutation via pre/post inspection of three CB state transitions. Generate node (C09) uses per-invocation provider lookup (never module singleton), asyncio-first awaited LLM calls, and automatic history accumulation via add_messages. Graph assembly (C10) uses a build_graph(checkpointer) factory pattern, hoists blocking I/O outside async generators, and surfaces astream_events(v2) results via SSE StreamingResponse. Graph smoke tests (C11) prove wiring with 14 fresh-MemorySaver tests to block state bleed.

**Commits 12–20** wire assessment and topic scoring into the graph. Assessment scaffold (C12) proves construction before LLM eval via try/except guard and named routing hooks. LLM assessment (C13) chains assessment_prompt | llm.with_structured_output(AssessmentOutput), with a prompt.__or__ patch pattern for LCEL tests. Topic scoring (C14) is a pure function merging delta, clamping to [0,1], and filtering via value-type (not allowlist). Adaptive prompts (C17) are five ChatPromptTemplate objects per mastery level with single {context} variable. Adaptive graph (C18) atomically updates prompt selection, cache key, and response schema; null-byte composite keys are injective; user_level None means assessment did not run. Profile UI (C19) nests @ui.refreshable over HTTP client, uses 280px sidebar + flex:1 chat, and renders all 6 topics as progress bars (defaulting to 0.0). Dynamic chat (C20) adds cycling stage labels, keeps sidebar live via profile_panel.refresh() in finally, surfaces user_level, and uses stage_active flag + finally ordering to prevent timer callback race.

**Key patterns:** Guard-clause auth (early closure, fail-safe), blocking-I/O hoisting (asyncio correctness), cache-key injection (composite, stable), add_messages reducer (automatic history), provider factory (LLM independence), state mutation avoidance (pure functions where possible), @ui.refreshable closure capture (NiceGUI asynchronous context binding), finally-block ordering (cleanup ordering).

**Structural constraints:** user_level drives phase gates and prompt selection, never written mid-graph to avoid circular updates. History excluded from LLM cache key (intentional gap). Assessment triggers at turn count or score threshold (pluggable in C23). User profiles in same SQLite file (transaction alignment). MemorySaver per-test prevents state bleed (test isolation).
